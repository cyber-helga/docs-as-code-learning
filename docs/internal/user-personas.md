Most documentation fails because it assumes “the user” is one person.

In reality, you likely have three to four distinct personas. 

## Business user (non-technical)
E-commerce manager
* Goal: get products live on marketplaces
* Pain: confusion, fear of breaking things
## Technical user
Developer / integrator
* Goal: connect APIs, automate flows
* Pain: missing details, unclear edge cases
## Operations/Support
Handles orders, returns, errors
* Goal: fix issues quickly
* Pain: hard-to-find troubleshooting steps
##  New customer (critical)
First-time user
* Goal: “get value fast”
* Pain: overwhelm
